# 园区级微电网运行监测平台 - 验收说明文档

## 一、验收概述

本文档定义了《园区级微电网运行监测平台》的验收标准、验收流程与测试用例。第三方可按照本清单独立完成部署验证与功能验收。

### 1.1 验收项目

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 基础设施 | Docker Compose 一键启动、所有服务正常运行 | P0 |
| 账号权限 | 登录、登出、密码修改、用户/角色管理、RBAC | P0 |
| 园区信息 | 园区新增、编辑、查询、删除 | P0 |
| 设备台账 | 设备新增、编辑、启用停用、多条件筛选查询 | P0 |
| 数据接入 | MQTT 与 HTTP 上报、数据校验、实时与历史存储 | P0 |
| 模拟数据 | 模拟器启停、持续产生演示数据、覆盖所有场景 | P0 |
| 监测大屏 | 运行总览、功率曲线、设备状态统计、园区切换 | P0 |
| 设备详情 | 基础信息、实时数据、24小时运行曲线 | P0 |
| 历史数据 | 多条件查询、图表展示、列表查看 | P1 |
| 日志审计 | 登录日志、操作日志记录与查询 | P1 |

### 1.2 验收环境准备

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux / Windows (需支持 Docker) |
| Docker | 24.0+ |
| Docker Compose | v2+ |
| 网络 | 可访问外网拉取镜像（首次部署） |
| 浏览器 | Chrome 100+ / Edge 100+ / Firefox 100+ |

## 二、验收用例清单

### TC-01: Docker Compose 一键启动

**前置条件**: 服务器已安装 Docker 和 Docker Compose

**步骤**:
1. 将项目代码完整复制到服务器
2. 复制 `.env.example` 为 `.env`，可使用默认配置
3. 执行 `docker compose up -d`
4. 等待 5-10 分钟（首次构建镜像耗时较长）
5. 执行 `docker compose ps`

**预期结果**:
- [ ] 所有 6 个服务（postgres, redis, emqx, backend, frontend, nginx）状态均为 `Up`
- [ ] postgres、redis、emqx 健康检查显示 `(healthy)`
- [ ] 无容器持续重启

---

### TC-02: 服务可访问性

**步骤**:
1. 浏览器访问 `http://<服务器IP>`
2. 访问后端接口 `curl http://<服务器IP>/api/auth/login -X POST -H "Content-Type: application/json" -d '{}'`

**预期结果**:
- [ ] 浏览器正常显示登录页（包含用户名、密码输入框）
- [ ] 后端接口返回 JSON 格式响应（非连接错误）

---

### TC-13: MQTT 数据接入验证

**步骤**:
1. 确保 TC-12 模拟器已启动
2. 查看后端日志: `docker compose logs -f backend`
3. 观察是否有数据接入日志
4. 或使用 MQTT 客户端连接 `tcp://<服务器IP>:1883`
5. 向主题 `microgrid/device/PV-001/data` 发送:
```json
{"timestamp":"2024-01-01T12:00:00","deviceCode":"PV-001","metricCode":"activePower","metricValue":500.5,"statusFlag":1}
```

**预期结果**:
- [ ] 后端日志显示接收到 MQTT 消息
- [ ] 非法数据（设备编码不存在、指标缺失）被拒绝并不入库
- [ ] 合法数据成功入库

---

### TC-14: HTTP 数据接入验证

**步骤**:
1. 使用 curl 或 Postman 调用:
```bash
curl -X POST http://<服务器IP>/api/data/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "data": [{
      "timestamp": "2024-01-01T12:00:00",
      "deviceCode": "PV-001",
      "metricCode": "activePower",
      "metricValue": 123.45,
      "statusFlag": 1
    }]
  }'
```
2. 发送缺少 deviceCode 的非法数据

**预期结果**:
- [ ] 合法数据返回 code=200
- [ ] 非法数据返回 code=400 或错误提示
- [ ] 非法数据未写入数据库

---

## 三、验收通过标准

### 3.1 评分规则

| 优先级 | 用例数 | 通过分值/用例 |
|--------|--------|--------------|
| P0 | 20 | 4 分 |
| P1 | 5 | 2 分 |
| 合计 | 25 | 满分 90 分 |

### 3.2 通过条件

- [ ] **必过条件**: 所有 P0 用例 100% 通过
- [ ] **总分条件**: 总分 ≥ 80 分（P1 用例最多可失败 5 个）
- [ ] **性能条件**:
  - 首屏加载时间 ≤ 3 秒（Chrome 网络面板，禁用缓存）
  - 模拟器启动后监测大屏数据刷新延迟 ≤ 5 秒
- [ ] **稳定性条件**:
  - 所有服务连续运行 1 小时无异常重启
  - 后端日志无 ERROR 级别异常（业务告警除外）
