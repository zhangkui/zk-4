# 园区级微电网运行监测平台 - 部署说明文档

## 一、系统架构

```
┌──────────────────────────────────────────────────────────┐
│                        Nginx (80)                         │
│              静态资源 + API 反向代理                        │
└───────────┬──────────────────────────────┬───────────────┘
            │                              │
     ┌──────▼──────┐              ┌────────▼────────┐
     │   Frontend  │              │    Backend      │
     │  Vue3 + Vite │              │ Spring Boot 3   │
     │   (静态文件)  │              │  Java 21 :8080  │
     └──────────────┘              └───┬─────────┬───┘
                                       │         │
                              ┌────────▼──┐  ┌──▼─────────┐
                              │ PostgreSQL │  │   Redis    │
                              │ +Timescale │  │  (缓存)    │
                              │  DB :5432  │  │   :6379    │
                              └───────────┘  └────────────┘
                                       │
                              ┌────────▼──────┐
                              │     EMQX       │
                              │  MQTT Broker   │
                              │   :1883        │
                              └────────────────┘
```

## 三、部署步骤

### 3.1 获取项目源码

将项目完整代码上传至服务器目标目录，例如 `/opt/microgrid`：

```
/opt/microgrid/
├── backend/           # 后端源码
├── frontend/          # 前端源码
├── deploy/            # 部署相关文件
│   ├── sql/           # 数据库初始化脚本
│   └── nginx/         # Nginx 配置
├── docker-compose.yml # 容器编排
├── .env.example       # 环境变量模板
└── docs/              # 文档
```

### 3.2 配置环境变量

复制环境变量模板并修改：

```bash
cd /opt/microgrid
cp .env.example .env
vi .env
```

`.env` 内容说明（**生产环境必须修改默认密码**）：

```env
# PostgreSQL
POSTGRES_DB=microgrid
POSTGRES_USER=microgrid
POSTGRES_PASSWORD=YourStrongPassword@2024   # 请修改为强密码
POSTGRES_PORT=5432

# Redis
REDIS_PASSWORD=YourRedisPassword@2024       # 请修改为强密码
REDIS_PORT=6379

# MQTT (EMQX)
MQTT_USERNAME=admin
MQTT_PASSWORD=YourMqttPassword@2024         # 请修改

# JWT (至少 256 bits，生产请使用随机长字符串)
JWT_SECRET=your-jwt-secret-key-at-least-32-characters-long-for-security-please-change

# 服务端口映射
BACKEND_PORT=8080
NGINX_PORT=80
```

### 3.3 启动所有服务

```bash
cd /opt/microgrid
docker compose up -d
```

首次启动会自动：
1. 拉取 PostgreSQL+TimescaleDB、Redis、EMQX、Nginx 基础镜像
2. 构建后端镜像（Maven 编译约需 3-5 分钟）
3. 构建前端镜像（npm build 约需 2-3 分钟）
4. 按依赖顺序启动各服务
5. PostgreSQL 自动执行 `deploy/sql/init.sql` 初始化数据库

### 3.4 查看服务状态

```bash
docker compose ps
```

预期所有服务状态均为 `Up` 或 `Up (healthy)`：

| 服务 | 容器名 | 端口 | 状态 |
|------|--------|------|------|
| postgres | microgrid-postgres | 5432 | Up (healthy) |
| redis | microgrid-redis | 6379 | Up (healthy) |
| emqx | microgrid-emqx | 1883, 18083 | Up (healthy) |
| backend | microgrid-backend | 8080 | Up |
| frontend | microgrid-frontend | - | Up |
| nginx | microgrid-nginx | 80 | Up |

### 3.5 查看日志

```bash
# 所有服务日志
docker compose logs -f

# 指定服务日志
docker compose logs -f backend
docker compose logs -f postgres
docker compose logs -f emqx
```

### 3.6 验证服务

1. 访问系统首页：`http://<服务器IP>:80`
2. 后端健康检查：`curl http://localhost:8080/api/auth/login`（应返回 401 或参数错误 JSON，而非连接拒绝）
3. EMQX 控制台：`http://<服务器IP>:18083`（默认账号 admin/public，建议立即修改）

## 五、启动模拟数据

系统部署完成后默认无真实硬件设备，需启动数据模拟器生成演示数据：

1. 浏览器登录系统（使用 admin 账号）
2. 左侧菜单进入 **数据模拟器**
3. 点击 **启动全部模拟器** 按钮
4. 返回监测大屏即可看到实时运行数据

模拟器覆盖场景：
- 光伏出力（日出高、日落低）
- 风电随机波动
- 园区负荷（早晚高峰）
- 储能充放电（根据净出力自动调节）
- 充电桩随机占用状态变化

## 六、常用运维命令

```bash
# 启动
docker compose up -d

# 停止
docker compose stop

# 停止并删除容器（保留数据卷）
docker compose down

# 停止并删除容器和数据卷（⚠️ 清空所有数据）
docker compose down -v

# 重启某个服务
docker compose restart backend
docker compose restart nginx

# 重新构建并启动（代码更新后）
docker compose up -d --build backend
docker compose up -d --build frontend

# 进入容器
docker exec -it microgrid-postgres psql -U microgrid -d microgrid
docker exec -it microgrid-redis redis-cli -a <密码>
docker exec -it microgrid-backend sh
```


## 八、常见问题

### Q1: 首次启动后端容器一直重启

**原因**: PostgreSQL 未就绪或数据库初始化失败

**排查**:
```bash
docker compose logs postgres
docker compose logs backend
```

确认 init.sql 已自动执行（首次启动时），检查 `.env` 中数据库密码是否与 application-docker.yml 中配置一致。

### Q2: 前端页面空白或 404

**原因**: 前端未正确构建或 Nginx 路由配置问题

**排查**:
```bash
docker compose logs nginx
docker exec microgrid-frontend ls /usr/share/nginx/html
```

若没有 `index.html`，尝试重新构建前端：
```bash
docker compose up -d --build frontend
```

### Q3: 监测大屏无数据

**原因**: 模拟器未启动或 MQTT 连接异常

**排查**:
1. 登录系统 → 数据模拟器 → 启动全部模拟器
2. 检查 EMQX 状态: `docker compose logs emqx`
3. 检查后端 MQTT 连接日志: `docker compose logs backend | grep -i mqtt`

### Q4: 端口被占用

修改 `.env` 中的端口映射：
```env
POSTGRES_PORT=15432   # 改为其他端口
REDIS_PORT=16379
BACKEND_PORT=18080
NGINX_PORT=8080
```

### Q5: 修改密码后不生效

修改 `.env` 后数据卷已存在不会重新初始化。需要：
1. 修改 `.env`
2. 若修改 DB 密码：同时进入 PostgreSQL 修改用户密码，或 `docker compose down -v` 重新初始化（丢失数据）

## 九、防火墙开放端口

| 端口 | 协议 | 用途 | 是否必须对公网开放 |
|------|------|------|--------------------|
| 80 | TCP | Nginx Web 服务 | ✅ 是 |
| 1883 | TCP | MQTT 设备接入 | 仅设备需要 |
| 8083 | TCP | MQTT WebSocket | 仅设备需要 |
| 8883 | TCP | MQTT SSL | 仅设备需要 |
| 5432 | TCP | PostgreSQL | ❌ 否（内部使用） |
| 6379 | TCP | Redis | ❌ 否（内部使用） |
| 8080 | TCP | Backend API | ❌ 否（Nginx 代理） |
| 18083 | TCP | EMQX 管理台 | 可选（运维用） |

> ⚠️ 生产环境建议仅开放 80 和 1883 端口，其他端口通过内部网络访问。
